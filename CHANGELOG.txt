Changelog
=========

2026-05-18 (v6.10.7)
Update-Manager: Hilfe-Menü → "Updates & Versionen…"

- Neuer Dialog im Hilfe-Menü (macOS + Windows): alle Releases werden über die
  öffentliche GitHub-API geladen — kein Token nötig.
- Zeigt alle verfügbaren Versionen mit Datum, Größe des Assets und ★ NEU / ✓ aktuell-Markierung.
- Changelog der gewählten Version wird rechts angezeigt.
- Download des passenden Assets (DMG auf macOS, ZIP auf Windows) in ~/Downloads
  mit Fortschrittsbalken.
- Nach Download: "DMG öffnen?" (macOS) bzw. "Im Explorer anzeigen" (Windows).
- Ältere Versionen können genauso heruntergeladen werden wie die neueste.

2026-05-18 (v6.10.6)
Windows: SDK-Demo-Dialog-Unterdrücker verbessert

- Demo-Dialog-Suppressor pollt jetzt alle 100 ms statt jede Sekunde — der Dialog
  verschwindet bevor er sichtbar oder hörbar wird.
- Prüft jetzt die Prozess-ID: nur Dialoge des eigenen Prozesses werden geschlossen,
  keine fremden Systemdialoge.
- Weitere Schlüsselwörter: "bearware", "trial", "license", "lizenz".
- Schließt mit WM_CLOSE + WM_COMMAND IDOK (Fallback) statt nur WM_COMMAND.
- Keine Änderung an macOS.

2026-05-18 (v6.10.5)
Streaming-Lautstärke: Standard von 100 auf 50 gesenkt, Minimum auf 10 erweitert

- Streaming-Lautstärke-Standard für alle Quellen (URL, yt-dlp, Webradio, Podcast, Playlist)
  von 100 auf 50 gesenkt (Faktor 0,5 = −6 dB). Webradio-Streams sind auf Sendepegel
  normalisiert und klangen auf Standard 100 % zu laut.
- Minimum von 25 auf 10 gesenkt für feinere Abschwächung sehr lauter Streams.
- Bereich-Anzeige aktualisiert: "(10–400, Standard: 50)".
- Gilt für wx (macOS) und Qt (Windows/Linux) gleichermaßen.

2026-05-18 (v6.10.4)
Vier weitere ttaccessible-Ideen: Per-Nutzer-Stereo-Balance, persistente Sitzungshistorie, Smart-Audio-Resolver

- Per-Nutzer-Stereo-Balance — über das Nutzermenü (Kanäle-Tab) kann für jeden Teilnehmer
  die Stereo-Position auf "Normal", "Nur links" oder "Nur rechts" gesetzt werden.
  Die Einstellung wird per Nutzername gespeichert und bei erneutem Beitritt automatisch angewandt.
- Persistente Sitzungshistorie — Verbindungen, Kanalwechsel sowie An- und Abmeldungen werden
  in einer JSONL-Datei (session_history.jsonl im App-Datenverzeichnis) dauerhaft protokolliert.
  Export als Textdatei über Datei → "Sitzungsverlauf exportieren…".
- Smart Audio Device Resolver — beim Auto-Initialisieren der Audiogeräte nach dem Login wird
  nun eingebaute Hardware (Built-in, Mikrofon, MacBook-Mikrofon) gegenüber virtuellen
  Loopback-Geräten bevorzugt, um versehentliche Stille zu vermeiden.

2026-05-17 (v6.10.3)
Vier weitere ttaccessible-Ideen: AVSpeechSynthesizer-Backend, TTS-Rate/Lautstärke für macOS, Benachrichtigungsmodus pro Typ, Root-Kanal-Auto-Join

- AVSpeechSynthesizer (PyObjC) — neues TTS-Backend "AVSpeechSynthesizer (PyObjC)".
  Nutzt Apples AVFoundation direkt via PyObjC — zuverlässiger als der say-Prozess,
  unterstützt Rate und Lautstärke pro Äußerung.
- TTS-Sprechtempo & Lautstärke für macOS-Backends — zwei neue Schieberegler im
  TTS-Bereich (nur macOS): "Sprechtempo (0–100)" und "Lautstärke (0–100)".
  Bei "macOS say" wird das Sprechtempo in WPM (50–400) umgerechnet; bei
  "AVSpeechSynthesizer" werden beide Parameter direkt gesetzt.
- Benachrichtigungsmodus pro Nachrichtentyp — die drei Häkchen für Hintergrund-
  Benachrichtigungen (Privatnachrichten / Kanalnachrichten / Rundnachrichten) sind
  nun Auswahlmenüs mit vier Modi: "Aus", "Benachrichtigung", "TTS", "VoiceOver".
  TTS und VoiceOver geben die Nachricht per Sprachausgabe aus, wenn die App nicht
  im Vordergrund ist.
- Root-Kanal-Auto-Join — neue Einstellung "Nach Anmeldung automatisch Root-Kanal
  beitreten" (Allgemein-Tab). Wenn aktiviert, wird nach jedem erfolgreichen Login
  automatisch der Root-Kanal betreten.

2026-05-17 (v6.10.2)
Vier neue Features von ttaccessible abgeleitet: granulare TTS-Toggles, macOS-Stimme, Jitter-Buffer, Aufnahmemodus

- TTS: Anmeldung/Abmeldung separat steuerbar — neues Häkchen "Anmeldung/Abmeldung" im
  TTS-Bereich. Server-Login/Logout-Ereignisse werden nun unabhängig von "System vorlesen" gesteuert.
- TTS: Kanal-Datei-Ereignis — neues Häkchen "Datei hinzugefügt/entfernt". Wenn eine Datei
  im aktuellen Kanal erscheint oder entfernt wird, wird der Dateiname via TTS angesagt.
- macOS TTS-Stimme (say-Befehl) — drittes TTS-Backend "macOS say (Stimme wählen)".
  Benutzt den macOS-`say`-Befehl mit einer wählbaren Stimme (Dropdown mit allen
  auf dem System verfügbaren Stimmen).
- Adaptiver Jitter-Buffer (SDK) — die bestehende Einstellung "Adaptiver Jitter-Buffer"
  (Einstellungen-Tab) wird jetzt tatsächlich auf SDK-Ebene angewendet (TT_SetUserJitterControl)
  wenn ein Nutzer dem Kanal beitritt.
- Aufnahmemodus — beim Start einer Aufnahme kann nun zwischen "Zusammengemischt (muxed)",
  "Pro Nutzer (separate)" und "Beides" gewählt werden. Modus wird gespeichert.

2026-05-17 (v6.10.1)
Zwei neue Features: Mediendatei-Stummschaltung (⌘Ctrl⇧M), Kanal-TTS-Stummschaltung

- Mediendatei-Stummschaltung (⌘Ctrl⇧M): Der ausgewählte Nutzer in der Nutzerliste kann
  mit diesem Tastenkürzel bezüglich seines Mediendatei-Audio-Streams stummgeschaltet oder

  "TTS-Meldungen stummschalten" bzw. "TTS-Meldungen entstummen". Stummgeschaltete Kanäle
  unterdrücken TTS-Ankündigungen für Beitritt und Verlassen von Nutzern.
  Die Einstellung wird dauerhaft gespeichert.

2026-05-17 (v6.10.0)
Drei neue Features: VoiceOver-TTS, Abwesend-Ankündigungen, ⌘K Schnellkanal-Wechsler

- VoiceOver-TTS-Backend (macOS): Im Tab "System & TTS" kann unter "TTS-Engine" zwischen
  espeak-ng und "VoiceOver (macOS)" gewählt werden. VoiceOver-Modus spricht Text direkt
  über die macOS-VoiceOver-Stimme (AppleScript-Schnittstelle), ohne espeak-ng.

- Schnellkanal-Wechsler (⌘K): Öffnet ein Suchfenster mit allen verfügbaren Kanälen.
  Tippen filtert die Liste in Echtzeit; Enter tritt dem ausgewählten Kanal bei.
